Reviewed against the accessible `Qs-KDK.pdf` v01.2, `Qs-GRZ.pdf` v01.4, and the vendored schemas. Main findings:

- **P0: `kdk-7` follow-up checking is effectively dead.** [kdk.rules.json](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/rules/kdk.rules.json:31) uses `followUp.followUpOds.0.followUpDate`, but `resolve()` only supports object keys and `[]`, not numeric indexes ([rules.py](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/rules.py:32)). No anchor is found, so follow-up date checks return silently ([rules.py](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/rules.py:207)). Also rare disease uses `followUpRds`, not `followUpOds` ([RareDiseasesFollowUp.json](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/schemas/kdk/RareDiseasesFollowUp.json:9)). Fix: split oncology/rare anchors and use wildcard-aware, per-follow-up comparison.

- **P0: `kdk-7` excludes too much.** The QS text excludes only `researchConsents/presentationDate` and `submission/date`; the rule also excludes every `/molecularBoardDecisionDate$` ([kdk.rules.json](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/rules/kdk.rules.json:36)). That skips plan and rare-diagnosis board dates such as `carePlanOd.molecularBoardDecisionDate` ([OncologyPlan.json](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/schemas/kdk/OncologyPlan.json:47)) and `diagnosisRd.molecularBoardDecisionDate` ([RareDiseasesCase.json](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/schemas/kdk/RareDiseasesCase.json:51)). Fix: remove that regex or exclude only the actual self-anchor when necessary.

- **P0: multi-scope MV consent is mis-modeled.** `kdk-9b` anchors on `metaData.mvConsent.scope[].date` ([kdk.rules.json](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/rules/kdk.rules.json:48)), but `dates_after()` takes only the first unfiltered scope date ([rules.py](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/rules.py:177)). `kdk-9a` compares against every scope date, including non-`mvSequencing` and `deny` entries. Fix: add an anchor primitive that filters `scope[]` by `type=permit` and `domain=mvSequencing`, then compare against the relevant consent date.

- **P1: `kdk-9b` mixes oncology and rare paths in one rule.** The absent branch paths are ignored, but this hides branch-specific semantics and granularity problems ([kdk.rules.json](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/rules/kdk.rules.json:49)). Split into OD/RD rules with branch guards. For RD, decide how to handle `followUpRds[].deathDate`, which is `YYYY-MM` in the schema ([RareDiseasesFollowUp.json](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/schemas/kdk/RareDiseasesFollowUp.json:74)), despite the QS text listing a day-granular date.

- **P1: rare-disease `libraryType` is encoded ambiguously.** `kdk-10` checks both `case.diagnosisOd.libraryType` and `case.diagnosisRd.libraryType` ([kdk.rules.json](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/rules/kdk.rules.json:66)), but the real rare path is `case.diagnosisRd.libraryType` ([RareDiseasesCase.json](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/schemas/kdk/RareDiseasesCase.json:65)). Fix: make this branch-aware and add a separate consistency rule requiring rare branch data to have `submission.diseaseType == "rare"`.

- **P1: KDK criterion 2 is only half implemented.** The rule checks LE-ID list membership ([kdk.rules.json](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/rules/kdk.rules.json:15)), but not admission for the submitted indication. Fix: config should carry OE/SE/hereditary admission flags and the primitive should check them against `metaData.submission.diseaseType`.

- **P1: required external checks can silently pass.** Missing config for KDK/GRZ node IDs or LE IDs yields `info`, which does not fail validation ([rules.py](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/rules.py:260)). For QS mode, missing required config should be a CLI error or rule error, not a pass.

- **P1: KDK criterion 4 is not enforced by this CLI path.** The rule file says terminology resolvability is elsewhere ([kdk.rules.json](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/rules/kdk.rules.json:7)), but `--kdk-rules` does not invoke such tooling ([cli.py](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/cli.py:36)). Either integrate it or stop claiming the semantic rule set covers all 12 criteria.

- **P1: KDK conditional completeness for `genomicDataCenterId` is missing.** The QS text requires it when sequencing was performed; schema does not require it ([Submission.json](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/schemas/kdk/Submission.json:60)). Add `required_if libraryType != "none"` for OD and RD.

- **P1: date validation is too weak.** `Draft202012Validator` is created without a `FormatChecker` ([validator.py](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/validator.py:193)), and `_date_tuple()` is loose/unanchored ([rules.py](/Users/kohlbach/Claude/MII/genomde-dk-validator/src/genomde_dk_validator/rules.py:51)). Fix: enable format checking, anchor parsing, and use `datetime.date.fromisoformat`.

Fix with terminology/code-set aware category mapping.
